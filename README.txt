/**
 * ================================================================
 * spotify-user-monitor.gs v2.0
 * ================================================================
 *
 * 특정 스포티파이 유저 프로필의 변경 사항을 주기적으로 감지하고
 * 디스코드 웹훅으로 알림을 전송하는 Google Apps Script입니다.
 *
 * 감지 항목:
 *   · 유저 표시 이름 변경
 *   · 공개 플레이리스트 추가 / 삭제
 *   · 플레이리스트 이름 변경
 *   · 플레이리스트 내 트랙 추가 / 삭제
 *
 * ── 설정 방법 ──────────────────────────────────────────────────
 * 1. GAS 에디터 → 프로젝트 설정 → 스크립트 속성에 아래 두 값 등록
 *    DISCORD_WEBHOOK_URL  │ 디스코드 채널 웹훅 URL
 *    TARGET_USER_ID       │ 감시할 스포티파이 유저 ID
 *    (프로필 링크 https://open.spotify.com/user/{ID} 에서 {ID} 부분)
 *
 * 2. monitorUser 함수를 원하는 주기(예: 5분)로 시간 기반 트리거 등록
 *
 * 3. 최초 1회 testSetup() 을 수동 실행하여 연결 확인 권장
 *
 * ── 알려진 한계 ─────────────────────────────────────────────────
 * · JS 실행 없이 서버사이드 렌더링 HTML만 파싱합니다.
 * · 프로필에 표시되는 플레이리스트가 약 10개를 초과하면 감지가
 *   불안정해질 수 있습니다.
 * · 스포티파이 HTML 구조 변경 시 파싱 로직 수정이 필요할 수 있습니다.
 *
 * ── v1 → v2 변경 내역 ──────────────────────────────────────────
 * [FIX-1] isFirstRun 판정 버그
 *          원인: displayName 갱신(STEP 1) 이후에 isFirstRun을 계산했기 때문에
 *                최초 실행 시 항상 false → 첫 실행임에도 모든 플리에
 *                "📂 새 플레이리스트 추가됨" 오알림 발생
 *          수정: displayName 갱신 전에 isFirstRun을 미리 계산
 *
 * [FIX-2] buildTrackChangeEmbed URL 버그
 *          원인: playlistUrl을 /embed/playlist/ 로 설정 →
 *                디스코드 링크 클릭 시 임베드 페이지로 이동
 *          수정: 직접 접근 URL /playlist/ 로 교체
 *
 * [FIX-3] HTML 엔티티 미디코딩
 *          원인: 이름에 &amp; &quot; 등이 포함된 경우 디코딩하지 않아
 *                저장된 이름과 새로 가져온 이름이 달라 이름 변경 오알림 발생
 *          수정: decodeHtml() 추가, 모든 이름 추출 시 일관 적용
 *
 * [FIX-4] UNKNOWN_NAME 오알림
 *          원인: 임베드 파싱 실패 시 "(이름 없음)" 반환 →
 *                기존 이름과 달라 이름 변경 오알림 발생
 *          수정: UNKNOWN_NAME 값은 이름 변경 감지 조건에서 명시적 제외
 *
 * [FIX-5] 제네릭 og:title 오인 (이름 변경 오알림의 핵심 원인)
 *          원인: 스포티파이 서버 점검/오류 시 og:title이 "Spotify" 등
 *                제네릭 값으로 반환 → 플레이리스트 이름으로 오인해 오알림
 *          수정: extractOgTitle()에서 "Spotify", "Open Spotify" 등
 *                제네릭 값 필터링 후 null 반환
 *
 * [FIX-6] 플레이리스트 이름 변경 즉시 알림
 *          원인: 이름 변경을 1턴만에 바로 알림 →
 *                순간적 서버 오류로 잘못된 이름이 반환될 때 오알림
 *          수정: pendingNameVerify 필드 추가,
 *                2턴 연속 동일한 이름 변경이 확인될 때만 알림 전송
 *
 * [ADD]   헬퍼 함수: testSetup(), resetState(), viewState()
 * [ADD]   상태 파일 버전 관리 및 마이그레이션 로직 (v1 → v2 자동 변환)
